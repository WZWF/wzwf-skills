# Impact Analysis — 输出模板

## 影响分析报告模板

```markdown
# 变更影响分析报告

> 目标: `{ClassName.methodName}`
> 风险等级: {HIGH/MEDIUM/LOW}
> 扫描范围: {N} 个类 | 深度: {depth}

## 一、影响概览

| 指标 | 数值 |
|------|------|
| 直接调用者 | {N} 个 |
| 间接调用者（深度2+） | {N} 个 |
| 被调用方法 | {N} 个 |
| Spring 隐式依赖 | {N} 个 |
| 循环调用 | {N} 个 |

## 二、调用链追踪

### 谁调用了目标？（Callers）

| 深度 | 类 | 方法 | 文件 | 行号 |
|------|-----|------|------|------|
| 1 | {Class} | {method} | {file} | {lines} |

### 目标调用了谁？（Callees）

| 深度 | 类 | 方法 | 说明 |
|------|-----|------|------|
| 1 | {Class} | {method} | {说明} |

## 三、Spring 隐式依赖

### AOP 切面
- [ ] {已确认/有风险/不适用}: {详情}

### 事件监听
- [ ] {已确认/有风险/不适用}: {详情}

### 定时任务
- [ ] {已确认/有风险/不适用}: {详情}

### 配置依赖
- [ ] {已确认/有风险/不适用}: {详情}

### 事务边界
- [ ] {已确认/有风险/不适用}: {详情}

### 缓存操作
- [ ] {已确认/有风险/不适用}: {详情}

### 异步标记
- [ ] {已确认/有风险/不适用}: {详情}

## 四、风险因素

{从 risk_assessment.factors 列出，每条附带代码证据}

## 五、变更建议

1. **安全变更范围**: {可以安全修改的部分}
2. **需要联动修改的文件**: {列出需要同步修改的文件和行号}
3. **需要验证的场景**: {列出需要测试覆盖的场景}
4. **不建议的操作**: {风险过高不建议做的改动}
```

## 简版报告模板（低风险场景）

当 `risk_assessment.level` 为 `LOW` 时使用简版：

```markdown
# 变更影响分析: `{target}`

**风险等级: LOW** ✅

调用者 {N} 个，均为同模块内部调用。无 AOP/事件/定时任务/事务依赖。

### 调用者清单
{列表}

### 变更建议
可安全修改，注意同步更新 {相关测试文件}。
```
