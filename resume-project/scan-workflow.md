# 身份识别与项目扫描

## 第零步：识别用户身份（多人项目必须）

本 skill 针对多人协作项目，必须先确定用户身份，再筛选其个人贡献。

### 1. 获取 git 身份

- 在当前子系统目录下执行 `git config user.name` 和 `git config user.email`
- 如果当前目录不是 git 仓库，尝试向上查找父目录的 `.git`
- 将结果展示给用户，询问："这是你吗？如果不是，请提供你的 git 用户名或邮箱"
- 如果既不是 git 仓库也找不到父级 `.git`，直接询问用户在项目中负责哪些模块
- 支持多身份：用户可能在不同时期用了不同的 git name/email（如 `czc` / `11051426`）

### 2. 分析个人提交记录

- 执行 `git log --author="<用户名>" --oneline --stat` 获取用户的提交及文件变更
- 如果用户有多个 git 身份，对每个身份都执行一次，合并结果
- 执行 `git log --author="<用户名>" --format="%H" | wc -l` 统计提交总数
- 执行 `git log --author="<用户名>" --pretty=format: --name-only | sort | uniq -c | sort -rn | head -30` 找出用户改动最多的文件
- **子系统场景特殊处理**：如果当前目录是子系统，git log 结果可能包含整个大仓库的记录，需要用 `-- <当前子系统路径>` 过滤

### 3. 定位个人负责模块

- 根据改动频率最高的文件/目录，推断用户负责的核心模块
- 对核心文件执行 `git blame` 确认用户的主要贡献行数
- 搜索 `@author` 注解匹配用户身份，作为辅助判断
- 将结果汇总为"我的模块清单"，展示给用户确认或补充
- 用户可能还参与了代码审查、方案设计等非代码工作，主动询问

### 4. 输出确认

- 向用户展示识别结果：
  ```
  根据 git 记录分析，你在 [当前子系统名] 中的主要贡献区域：
  - 模块/包名1（xx 个文件，xx 次提交）
  - 模块/包名2（xx 个文件，xx 次提交）
  - ...
  请确认或补充你负责的其他内容。
  ```
- 用户确认后再进入下一步

## 第一步：项目全景扫描（聚焦个人模块）

使用 Glob 和 Grep 工具分析项目，**重点聚焦第零步识别出的个人负责模块**。

**子系统场景适配**：当用户只打开了一个子系统时，扫描范围就是当前目录。如果需要扫描其他子系统目录，使用 Shell 工具的 `working_directory` 参数切换到对应路径执行命令。

### 1. 技术栈识别

- 读取当前目录的 `pom.xml` / `build.gradle` 识别框架和依赖版本
- 如果是子模块，还应读取父 `pom.xml`（通过 `<parent>` 或 `<relativePath>` 定位）获取统一版本管理
- 搜索 `application.yml` / `application.properties` / `bootstrap.yml` 获取配置信息
- 识别 Spring Boot 版本、数据库、中间件、消息队列等
- 检查是否有 `spring.cloud.nacos` / `spring.cloud.gateway` 等微服务配置，判断整体架构

### 2. 项目结构分析

- 扫描目录结构，识别模块划分
- 对于典型的 `xxx-api` + `xxx-server` 结构：api 是对外接口定义，server 是实现
- 识别分层架构（controller / service / dao / config / module 等）
- 注意识别非标准分层（如按业务域划分的 module 包结构）

### 3. 个人业务功能提取（重点）

- 重点分析用户负责的 Controller 接口和 Service 实现
- 梳理用户模块的业务逻辑和核心功能
- 识别用户代码中的自定义注解、AOP 切面等技术亮点
- 检查是否有独立设计的引擎/框架类代码（Registry/Factory/Strategy/Hook 等命名）

### 4. 个人技术亮点挖掘（重点）

- 在用户负责的模块中搜索设计模式应用
- 在用户代码中识别缓存、并发、性能优化等技术实践
- 分析用户提交信息中的关键词（fix/optimize/refactor/add）挖掘优化点
- 从 commit message 中提取用户解决的问题和技术方案
- 检查是否有 Starter / SDK / 公共组件等基础设施贡献（可能在其他目录中）
