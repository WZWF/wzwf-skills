# Resume Project

提炼当前 Java 项目为简历项目经历，输出可直接用于简历的项目描述。

## 何时主动使用

1. 用户提到简历/求职 → 主动提议分析当前项目
2. 用户打开 Java 项目并说「整理项目亮点」→ 启动完整流程
3. 用户问「怎么描述这个项目」→ 结构化输出
4. 面试准备 → 生成口述版 + 追问准备

## 适用场景

- 用户想把当前项目整理成简历中的项目经历
- 用户需要提炼项目亮点、技术难点、个人职责

## 依赖与前置

### 工具依赖

| 工具 | 用途 | 必须 |
|------|------|------|
| Shell | 执行 git 命令（log/blame/config）、切换目录 | ✅ |
| Glob | 扫描文件结构（pom.xml/yml/java文件） | ✅ |
| Grep | 搜索代码内容（@author/注解/设计模式关键词） | ✅ |
| Read | 读取文件内容（pom.xml/源码/配置文件） | ✅ |
| Feedback MCP | 与用户交互确认（身份/模块/呈现方式选择） | ✅ |
| WebSearch | 搜索目标岗位 JD、行业关键词对标（第五步） | 可选 |

### 关联 Skill

| Skill | 关系 | 说明 |
|-------|------|------|
| `brainstorming` | 可前置调用 | 如果用户不确定想突出什么亮点，可先进入头脑风暴探索 |
| `sonar-fix` | 辅助参考 | 项目的代码质量数据可作为简历中"注重代码质量"的佐证 |
| `database-operations` | 辅助参考 | 如果项目涉及数据库设计/优化，可查看表结构辅助描述 |

### 前置条件

1. 当前工作区或用户提供的路径中有 Java 项目代码
2. 有 git 历史记录（如无 git 则需用户口述个人贡献）
3. 用户能确认自己的 git 身份（name/email/@author）

## 执行流程

| 阶段 | 摘要 |
|------|------|
| **入口** | 不假设工作区即目标项目；判断 A/B 上下文 → 确定 target_dirs → 选择单目录/多子系统聚合/多独立项目模式 |
| **第 0.5 步** | 识别子系统边界，让用户选择呈现方式（大项目+模块 / 拆分多条 / 挑亮点） |
| **第零步** | 获取 git 身份 → 分析个人提交 → 定位负责模块 → 用户确认 |
| **第一步** | 聚焦个人模块：技术栈、结构、业务功能、技术亮点 |
| **第二步** | 按 STAR-X 模板提炼核心贡献（3-5 条），体现 3 年中级定位 |
| **第三步** | STAR-X 润色、量化、动词、L3+ 深度、反模式与 ATS 关键词 |
| **第四步** | 输出完整版 / 精简版 / 口述版 |
| **第五步** | 简历优化建议、JD 匹配、面试追问方向 |
