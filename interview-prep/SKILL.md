# 面试题生成器（Interview Prep）

基于项目**真实代码**生成深度面试题，每道题必须能定位到具体类/方法，参考答案必须从代码中提取证据，禁止泛泛八股。

## 何时主动使用

- 用户提到：面试准备、面试题、模拟面试、项目深挖、八股文、技术深度
- 用户打开 Java/Spring Boot 项目并询问「可能被问什么」「怎么讲这个项目」
- 用户完成简历整理后需要配套面试追问（可配合 `resume-project` skill）

## 核心原则

| 原则 | 要点 |
|------|------|
| 代码优先 | 先读代码再出题，无证据不出题 |
| 深度优先 | 10 道深题 > 30 道浅题 |
| 追问链 | 每题 L1→L2→L3 三层递进 |
| 中文交互 | 中文表述，技术术语保留英文 |

## 工具依赖

| 工具 | 用途 |
|------|------|
| Glob | 扫描 pom.xml、Java 源码、Lua 脚本、配置文件 |
| Grep | 搜索注解、AOP 切面、Redis/Lua 关键词 |
| Read | 深度阅读核心类、配置、脚本、文档 |
| SemanticSearch | 定位设计决策、扩展点、异常处理逻辑 |

## 执行流程

| 步骤 | 摘要 |
|------|------|
| Step 1 | 扫描项目结构（Glob pom/Java/Lua/配置）→ 输出 5-10 行「项目画像」供用户确认 |
| Step 2 | 按 P0→P2 优先级深度阅读核心代码，边读边记设计决策 / 亮点 / 风险 |
| Step 3 | 阅读 README/CHANGELOG/docs，代码与文档冲突时以代码为准 |
| Step 4 | 整理内部「出题素材表」（选型理由、底层机制、防御逻辑、扩展接口、代码隐患） |
| Step 5 | 按 5 维度生成 10-15 题（设计决策 / 原理深挖 / 边界情况 / 扩展方向 / 陷阱） |
| Step 6 | 每题生成 3 层递进追问链：L1 确认理解 → L2 原理细节 → L3 极端场景 |
| Step 7 | 按 [templates.md](templates.md) 模板输出结构化 Markdown 面试题集 |
