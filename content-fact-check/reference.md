# 详细验证方法

## 学术论文验证

### arxiv 论文

1. 访问 `https://arxiv.org/abs/{论文ID}`（如 `2604.25850`）
2. 核对清单：
   - [ ] 论文标题与文稿引述一致
   - [ ] 作者/机构与文稿归属一致（注意：不要把 A 机构的论文归给 B 机构）
   - [ ] 发表日期（年/月）准确
   - [ ] 关键结论的引述是否忠实原文（不夸大、不断章取义）
   - [ ] 具体数据点（如精度 33.7%）在原文中可找到

### 常见陷阱

| 陷阱 | 说明 |
|------|------|
| 机构归属错误 | 复旦的论文被说成 Anthropic 的 |
| 数据点移花接木 | 把论文 A 的数据用来支撑论文 B 的结论 |
| 过度概括 | 论文说"某场景下"，文稿说"所有场景" |
| 版本混淆 | 引用了 v1 的结论，论文已更新到 v3 |

## 人物归属验证

### 头衔时效性

技术人物的头衔经常变化：

| 人物 | 常见错误 | 正确处理 |
|------|---------|---------|
| Andrew Ng | "Stanford 教授" | DeepLearning.AI 创始人、斯坦福兼职教授 |
| Andrej Karpathy | "OpenAI 研究员" | 前 OpenAI/Tesla，现独立研究者 |

**验证步骤**：
1. 查看个人官网或 LinkedIn 当前身份
2. 如果引述了观点，找到原始推文/文章/演讲
3. 标注"此观点出自其 XX 系列阐述"而非伪造单一论文来源

### 引述准确性

- **直接引用**：必须是原文，用引号标注
- **转述/总结**：不加引号，但注明"大意是"或"其观点认为"
- **综合多处**：注明"出自他在 XX 和 YY 中的系列阐述"

## GitHub 仓库验证

### API 查询

```bash
# 单个仓库
curl -s https://api.github.com/repos/owner/repo | jq '{name, stargazers_count, description, archived}'

# 批量（使用配套脚本）
python scripts/verify-github-repos.py repos.txt
```

### star 数表述规范

| 实际 star | 推荐写法 | 避免 |
|-----------|---------|------|
| 151,234 | 151k+ | 15 万多（过于模糊）/ 151,234（过于精确会过时） |
| 3,456 | 3.4k+ | 3k（损失精度）/ 3456（过于精确） |
| 87 | ~90 | 87（太精确）/ 100+（虚高） |

### 核对清单

- [ ] 仓库是否存在（未被删除/改名）
- [ ] 仓库是否被归档（archived=true 需标注）
- [ ] star 数量级是否正确（千级/万级/十万级）
- [ ] 文稿中的描述是否与仓库实际 README/description 一致
- [ ] 推荐原因是否合理（不是只看 star 数）

## 术语定义验证

### 验证层级

```
1. 标准制定方（如 agentskills.io）→ 最权威
2. 主流工具方（如 Cursor/Anthropic 官方文档）→ 高可信
3. 知名个人（如 Karpathy 的推文）→ 需标注为个人观点
4. 技术博客/媒体 → 需交叉验证
```

### 术语演化追踪

技术术语可能随时间演化：

```
Prompt Engineering (2022-2024)
  → Context Engineering (2025, Karpathy)
    → Harness Engineering (2026, David Daniel / AHE)
```

验证要点：
- 确认每个阶段的代表人物/来源
- 确认时间线是否准确
- 确认"替代"还是"包含"关系

## 技术事实验证

### 交叉验证标准

| 可信度 | 要求 |
|--------|------|
| 高 | 2+ 独立来源确认 |
| 中 | 1 个权威来源 + 合理推断 |
| 低 | 仅 1 个非权威来源 |

### 数据时效性标注

对于会快速变化的数据，在 references.md 中标注验证日期：

```markdown
| 数据点 | 值 | 验证日期 | 来源 |
|--------|-----|---------|------|
| anthropics/skills stars | 151k+ | 2026.06 | GitHub API |
| Skill 支持平台数 | 30+ | 2026.06 | agentskills.io |
```
